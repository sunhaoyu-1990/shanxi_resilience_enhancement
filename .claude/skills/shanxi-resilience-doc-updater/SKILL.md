# 陕交控项目文档更新工具

专门用于陕交控多路段改扩建韧性提升项目的文档维护工作。

## 核心功能

### 1. 会话记录更新
自动更新 `docs/会话记录与搭建过程.md`：
- 添加新的会话阶段
- 记录已完成的工作
- 更新项目状态清单
- 记录遇到的问题和解决方案

### 2. 业务逻辑记录
维护 `docs/业务逻辑记录.md`：
- 添加新的业务逻辑条目（BL-00x）
- 记录业务描述、处理逻辑
- 提供伪代码和 SQL 示例
- 关联相关表和字段

### 3. 表口径登记
更新表口径总表：
- 登记新创建的表
- 记录表用途、粒度、主键
- 说明来源和下游使用模块
- 关联字段字典

### 4. 项目状态维护
更新 `.claude/memory/MEMORY.md`：
- 记录项目当前状态
- 跟踪已完成的工作
- 列出待完成任务
- 记录关键决策和架构选择

## 使用示例

### 更新会话记录
```bash
skill: shanxi-resilience-doc-updater
args: "session --stage '收费路段数据导入' --step '数据分析与导入'"
```

### 添加业务逻辑
```bash
skill: shanxi-resilience-doc-updater
args: "business-logic --name '判断是否交控集团' --module M0"
```

### 登记表口径
```bash
skill: shanxi-resilience-doc-updater
args: "table-registry --table dim_toll_road --layer dim"
```

### 更新项目记忆
```bash
skill: shanxi-resilience-doc-updater
args: "memory --status '完成收费路段数据导入'"
```

## 数据表说明.md 更新规范

新建表后必须同步更新 `docs/数据表说明.md`，具体步骤：

1. **更新顶部日期**（第3行 `> 更新日期：YYYY-MM-DD`）

2. **更新表清单总览**（按字母序插入新行）：
```markdown
| [新表名](#N-新表名) | DIM/DWD/DWS/ADS | 简介 | 记录数 |
```
并将后续表的锚点编号依次 +1。

3. **插入详细章节**（在对应位置插入）：
```markdown
### N. 新表名

**简介**：...

**主键**：...

#### 数据字典

| 字段名 | 类型 | 可空 | 描述 |
|--------|------|------|------|

#### 查询示例

```sql
SELECT ...
```
```

4. **后续章节标题编号** 依次 +1。

## 自动更新触发点

### 数据导入后
- 更新会话记录
- 登记表口径
- 更新项目记忆

### 表结构变更后
- 更新表口径总表
- 更新字段字典
- 记录变更原因

### 业务逻辑实现后
- 添加业务逻辑记录
- 提供 SQL 示例
- 关联相关表

### 阶段完成后
- 更新会话记录
- 更新已完成工作清单
- 列出下一步任务
