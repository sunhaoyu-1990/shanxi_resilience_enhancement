### 1. dim_construction_segment

**简介**：施工区间维表，记录各工程方案下每个施工区间的基本属性，包括区间端点、施工方向、占用车道数、施工时长与开始时间。`construction_end_time` 为数据库自动计算列。

**主键**：`id`（自增）

**唯一约束**：`project_id + scheme_id + segment_start_point + segment_end_point + construction_direction`

#### 数据字典

| 字段名 | 类型 | 可空 | 描述 |
|--------|------|------|------|
| id | integer | NOT NULL | 自增主键 |
| project_id | varchar(20) | NOT NULL | 工程 ID |
| scheme_id | varchar(20) | NOT NULL | 方案 ID，同一工程可有多个方案 |
| segment_start_point | varchar(50) | NOT NULL | 施工区间起点名称 |
| segment_end_point | varchar(50) | NOT NULL | 施工区间终点名称 |
| construction_direction | integer | NOT NULL | 施工方向：1-上行，2-下行，3-双向 |
| lane_occupancy_count | integer | NOT NULL | 施工占用车道数，-1 表示占用全部车道 |
| construction_duration_days | integer | NOT NULL | 施工时长，单位：天 |
| construction_start_time | date | NULL | 施工开始时间 |
| construction_end_time | date | NULL | 施工结束时间（生成列，由开始时间 + 时长自动计算） |
| restricted_vehicle_types | varchar(200) | NULL | 限制车型，1~10对应1~10型车，"|"分隔多车型，"-1"表示全车型限制 |
| created_at | timestamp | NULL | 记录创建时间 |
| updated_at | timestamp | NULL | 记录更新时间 |

#### 查询示例

```sql
-- 查询所有施工区间
SELECT
    id,
    project_id,
    scheme_id,
    segment_start_point || ' → ' || segment_end_point AS segment,
    CASE construction_direction
        WHEN 1 THEN '上行'
        WHEN 2 THEN '下行'
        WHEN 3 THEN '双向'
    END AS direction,
    CASE lane_occupancy_count WHEN -1 THEN '全部车道' ELSE lane_occupancy_count::text || '条' END AS lanes,
    construction_start_time,
    construction_end_time,
    construction_duration_days || '天' AS duration,
    restricted_vehicle_types
FROM dim_construction_segment
ORDER BY construction_start_time, id;

-- 查询指定方案下的施工区间
SELECT * FROM dim_construction_segment
WHERE project_id = '1' AND scheme_id = '1'
ORDER BY construction_start_time;

-- 查询施工时间段与指定日期重叠的区间
SELECT * FROM dim_construction_segment
WHERE construction_start_time <= '2026-07-01'
  AND construction_end_time   >= '2026-07-01';
```

### 2. dim_detour_ratio_rule

**简介**：绕行通行费增幅区间与绕行比例对应规则表，用于 M5 通行费影响测算模块。增幅区间采用左闭右开原则，绕行比例表示该增幅下选择绕行的车辆占比。

**主键**：`id`

#### 数据字典

| 字段名 | 类型 | 可空 | 描述 |
|--------|------|------|------|
| id | varchar(20) | NOT NULL | 规则 ID（主键） |
| detour_toll_increase_min | float8 | NOT NULL | 绕行通行费增幅下限（含），取值 [0, 1] |
| detour_toll_increase_max | float8 | NOT NULL | 绕行通行费增幅上限（不含），取值 (0, 1] |
| detour_ratio | float8 | NOT NULL | 对应绕行比例，取值 [0, 1] |
| remark | text | NULL | 备注 |
| created_at | timestamp | NULL | 记录创建时间 |
| updated_at | timestamp | NULL | 记录更新时间 |

#### 当前规则数据

| 增幅区间 | 绕行比例 |
|----------|----------|
| 0% ~ 20% | 50% |
| 20% ~ 40% | 40% |
| 40% ~ 60% | 30% |
| 60% ~ 80% | 20% |
| 80% ~ 100% | 0% |

#### 查询示例

```sql
-- 查看所有规则
SELECT
    id,
    to_char(detour_toll_increase_min * 100, 'FM990.0') || '%' AS increase_min,
    to_char(detour_toll_increase_max * 100, 'FM990.0') || '%' AS increase_max,
    to_char(detour_ratio * 100, 'FM990.0') || '%' AS detour_ratio
FROM dim_detour_ratio_rule
ORDER BY detour_toll_increase_min;

-- 根据实际增幅比例匹配对应绕行比例
-- 例如：增幅为 35%（0.35）
SELECT detour_ratio
FROM dim_detour_ratio_rule
WHERE detour_toll_increase_min <= 0.35
  AND detour_toll_increase_max >  0.35;
```

### 3. dim_lane_capacity_rule

**简介**：各设计时速下单车道高速通行能力规则表，用于 M1 通行能力评估模块。根据路段设计时速查找对应的单车道通行能力基准值（pcu/h），再结合可用车道数计算路段总通行能力。

**主键**：`id`

**唯一约束**：`design_speed_kmh`

#### 数据字典

| 字段名 | 类型 | 可空 | 描述 |
|--------|------|------|------|
| id | varchar(20) | NOT NULL | 规则 ID（主键） |
| design_speed_kmh | float8 | NOT NULL | 设计时速，单位 km/h（唯一） |
| single_lane_traffic_capacity | float8 | NOT NULL | 单车道通行能力，单位 pcu/h |
| remark | text | NULL | 备注 |
| created_at | timestamp | NULL | 记录创建时间 |
| updated_at | timestamp | NULL | 记录更新时间 |

#### 当前规则数据

| 设计时速（km/h） | 单车道通行能力（pcu/h） |
|------------------|------------------------|
| 120 | 2200 |
| 100 | 2100 |
| 80 | 2000 |
| 60 | 1800 |

#### 查询示例

```sql
-- 查看所有规则
SELECT id, design_speed_kmh, single_lane_traffic_capacity
FROM dim_lane_capacity_rule
ORDER BY design_speed_kmh DESC;

-- 根据设计时速查询单车道通行能力
SELECT single_lane_traffic_capacity
FROM dim_lane_capacity_rule
WHERE design_speed_kmh = 120;

-- 计算路段总通行能力（单车道能力 × 可用车道数）
SELECT
    r.design_speed_kmh,
    r.single_lane_traffic_capacity,
    r.single_lane_traffic_capacity * 2 AS capacity_2_lanes,
    r.single_lane_traffic_capacity * 3 AS capacity_3_lanes
FROM dim_lane_capacity_rule r
ORDER BY design_speed_kmh DESC;
```

| 收费路段名称 | varchar(100) | NULL | 收费路段名称 |
| 路段性质 | varchar(10) | NULL | **还贷性**（交控集团）/ 经营性 |
| 路段类型 | varchar(10) | NULL | 高速路段 / 普通路段 |
| 联网收费状态 | varchar(10) | NULL | 联网收费 / 独立收费 |
| 路段里程 | numeric | NULL | 路段里程（米） |
| 起始计费位置经度 | numeric | NULL | 起始计费位置经度 |
| 起始计费位置纬度 | numeric | NULL | 起始计费位置纬度 |
| 起始计费位置桩号 | varchar(20) | NULL | 起始计费位置桩号 |
| 终止计费位置桩号 | varchar(20) | NULL | 终止计费位置桩号 |
| 终止计费位置经度 | numeric | NULL | 终止计费位置经度 |
| 终止计费位置纬度 | numeric | NULL | 终止计费位置纬度 |
| 是否纳税 | varchar(5) | NULL | 是 / 否 |
| 税率 | numeric | NULL | 税率 |
| 所属业主编号 | varchar(20) | NULL | 所属业主编号 |
| 路段收费方式 | varchar(10) | NULL | 封闭式 / 开放式 |
| 重合收费公路编号 | varchar(20) | NULL | 重合收费公路编号 |
| 开工时间 | date | NULL | 开工时间 |
| 通车时间 | date | NULL | 通车时间 |
| 停止收费时间 | date | NULL | 停止收费时间 |
| 待用税率 | numeric | NULL | 待用税率 |
| 待用税率生效时间 | date | NULL | 待用税率生效时间 |
| source_flag | varchar(16) | NULL | 数据来源标识 |
| created_at | timestamp | NULL | 记录创建时间 |
| updated_at | timestamp | NULL | 记录更新时间 |

#### 查询示例

```sql
-- 判断收费单元是否属于交控集团（取ID前11位匹配）
SELECT
    sp.id AS section_id,
    sp.name AS section_name,
    tr.收费路段编号,
    tr.路段性质,
    CASE tr.路段性质 WHEN '还贷性' THEN TRUE ELSE FALSE END AS is_jiaokong
FROM dwd_section_path sp
LEFT JOIN dim_toll_road tr
    ON LEFT(sp.id, 11) = tr.收费路段编号
WHERE sp.version_yyyymm = '202501'
LIMIT 20;

-- 统计路段性质分布
SELECT 路段性质, COUNT(*) AS cnt FROM dim_toll_road GROUP BY 路段性质;
```

### 11. dwd_tom_network_edges

**简介**：pgRouting 路网边表，由 `dwd_tom_noderelation` 转换而来，供 pgRouting 算法直接使用。`source` / `target` 为 pgRouting 内部整型节点 ID（对应 `dwd_tom_network_vertices.id`），`cost` 为正向代价（里程），`reverse_cost = 1e9` 表示单向路网。

**主键**：`id`

#### 数据字典

| 字段名 | 类型 | 可空 | 描述 |
|--------|------|------|------|
| id | bigint | NOT NULL | pgRouting 边 ID（主键） |
| source | bigint | NOT NULL | 起点节点 ID（pgRouting 内部） |
| target | bigint | NOT NULL | 终点节点 ID（pgRouting 内部） |
| cost | float8 | NOT NULL | 正向代价（里程，米） |
| reverse_cost | float8 | NOT NULL | 反向代价（单向路网设为 1e9） |
| version_yyyymm | varchar(6) | NOT NULL | 路网版本年月 |
| original_enroadnodeid | varchar(32) | NULL | 原始入口节点编号 |
| original_exroadnodeid | varchar(32) | NULL | 原始出口节点编号 |

| exroadnodetype | integer | NULL | 出口节点类型 |
| enroadnodename | varchar(100) | NULL | 入口节点名称 |
| exroadnodename | varchar(100) | NULL | 出口节点名称 |

```sql
-- 使用 pgRouting 计算最短路径（需知道 source/target 的整型节点ID）
SELECT seq, node, edge, cost, agg_cost

    'SELECT id, source, target, cost, reverse_cost FROM dwd_tom_network_edges WHERE version_yyyymm = ''202501''',
    起点节点ID,

FROM dwd_tom_network_vertices v
```

### 12. dwd_tom_network_vertices

**简介**：pgRouting 路网节点映射表，记录 pgRouting 内部整型节点 ID 与原始字符串节点 ID（`enRoadNodeId` / `exRoadNodeId`）之间的对照关系，是通过业务节点编号进行路径查询的关键索引表。

**主键**：`id`

#### 数据字典

| 字段名 | 类型 | 可空 | 描述 |
|--------|------|------|------|
| id | bigint | NOT NULL | pgRouting 节点 ID（主键） |
| original_node_id | varchar(32) | NOT NULL | 原始节点 ID（业务编号） |
| version_yyyymm | varchar(6) | NOT NULL | 路网版本年月 |
| node_type | integer | NULL | 节点类型 |
| node_name | varchar(100) | NULL | 节点名称 |
| created_at | timestamp | NULL | 记录创建时间 |

#### 查询示例

```sql
-- 通过业务节点 ID 查询 pgRouting 内部 ID
SELECT id AS pgr_node_id, node_name
FROM dwd_tom_network_vertices
WHERE original_node_id = '原始节点编号'
  AND version_yyyymm = '202501';

-- 完整路径查询示例：从业务节点编号 → 路径结果 → 还原节点名称
WITH start_node AS (
    SELECT id FROM dwd_tom_network_vertices
    WHERE original_node_id = '起点业务ID' AND version_yyyymm = '202501'
),
end_node AS (
    SELECT id FROM dwd_tom_network_vertices
    WHERE original_node_id = '终点业务ID' AND version_yyyymm = '202501'
),
path AS (
    SELECT seq, node, edge, cost, agg_cost
    FROM pgr_dijkstra(
        'SELECT id, source, target, cost, reverse_cost
         FROM dwd_tom_network_edges WHERE version_yyyymm = ''202501''',
        (SELECT id FROM start_node),
        (SELECT id FROM end_node),
        directed => true
    )
)
SELECT
    p.seq,
    v.original_node_id,
    v.node_name,
    p.cost      AS step_cost_m,
    p.agg_cost  AS total_cost_m
FROM path p
JOIN dwd_tom_network_vertices v
    ON p.node = v.id AND v.version_yyyymm = '202501'
ORDER BY p.seq;
```

### 15. dws_section_od_path_flow_hour

**简介**：收费单元-OD(path)小时车型流量统计表，按收费单元×OD-path映射×小时×车型粒度统计通行流量。数据源为CSV通行记录文件（如 `gstx_exit_with_min_fee202603.csv`），经过 intervalgroup+intervaltimegroup 同步修复、numpath映射、去重计数后聚合写入。同一记录内，同一收费单元在同一小时同一车型下只计1次流量。

**粒度**：`section_id × od_section_path_id × stat_hour × vehicle_type`

**主键**：`id`（自增）/ **业务唯一键**：`(section_id, od_section_path_id, stat_hour, vehicle_type)`

**建表 DDL**：`sql/ddl/dws/create_dws_section_od_path_flow_hour.sql`

**字段说明**：

| 字段 | 类型 | 可空 | 说明 |
|------|------|------|------|
| id | BIGSERIAL | NOT NULL | 自增主键 |
| section_id | VARCHAR(64) | NOT NULL | 收费单元ID，来自修复后 intervalgroup |
| od_section_path_id | BIGINT | NOT NULL | OD-Section-Path映射表ID，外键关联 dwd_od_section_path_map.id |
| stat_hour | TIMESTAMP | NOT NULL | 统计小时（带日期），如 2026-03-01 12:00:00 代表12:00~13:00的流量，取自修复后 intervaltimegroup 中该收费单元对应时间截断到小时 |
| vehicle_type | VARCHAR(32) | NOT NULL | 车型编码，取自feevehicletype(非空时)或envehicletype，均为空时为"0" |
| flow_cnt | INTEGER | - | 通行流量（自然车辆数），默认0 |
| source_flag | VARCHAR(32) | - | 数据来源标识，默认 `computed` |
| created_at | TIMESTAMP | - | 记录创建时间 |
| updated_at | TIMESTAMP | - | 记录更新时间 |

**约束**：
- `uq_dws_section_od_path_flow_hour`：UNIQUE (section_id, od_section_path_id, stat_hour, vehicle_type)
- `chk_flow_non_negative`：CHECK (flow_cnt >= 0)

**索引设计**（按三大查询模式优化）：

| 索引名 | 列 | 覆盖查询 |
|--------|-----|----------|
| idx_sopfh_section_odpath | (section_id, od_section_path_id, vehicle_type) | Q1: 查询收费单元影响的所有OD(path)（含车型） |
| idx_sopfh_odpath_hour | (od_section_path_id, stat_hour, vehicle_type) | Q2: 查询OD(path)的时间范围流量（含车型） |
| idx_sopfh_section_hour | (section_id, stat_hour, vehicle_type) | Q3: 查询收费单元的时间范围流量（含车型） |

**查询示例**：

```sql
-- Q1: 查询某收费单元影响的所有OD(path)
SELECT DISTINCT m.enid, m.exid, m.numpath
FROM dws_section_od_path_flow_hour f
JOIN dwd_od_section_path_map m ON f.od_section_path_id = m.id
WHERE f.section_id = 'G007061003000210'
ORDER BY m.enid, m.exid;

-- Q2: 查询某OD(path)的多日/小时流量
SELECT stat_hour, SUM(flow_cnt) AS total_flow
FROM dws_section_od_path_flow_hour
WHERE od_section_path_id = 42
  AND stat_hour BETWEEN '2026-03-01 00:00:00' AND '2026-03-31 23:59:59'
GROUP BY stat_hour
ORDER BY stat_hour;

-- Q2变体: 按日汇总
SELECT DATE(stat_hour) AS stat_day, SUM(flow_cnt) AS daily_flow
FROM dws_section_od_path_flow_hour
WHERE od_section_path_id = 42
  AND stat_hour >= '2026-03-01'
GROUP BY DATE(stat_hour)
ORDER BY stat_day;

-- Q3: 查询某收费单元的多日/小时流量
SELECT stat_hour, flow_cnt
FROM dws_section_od_path_flow_hour
WHERE section_id = 'G007061003000210'
  AND stat_hour BETWEEN '2026-03-15 00:00:00' AND '2026-03-15 23:59:59'
ORDER BY stat_hour;

-- Q3变体: 按日汇总
SELECT DATE(stat_hour) AS stat_day, SUM(flow_cnt) AS daily_flow
FROM dws_section_od_path_flow_hour
WHERE section_id = 'G007061003000210'
  AND stat_hour >= '2026-03-01'
GROUP BY DATE(stat_hour)
ORDER BY stat_day;
```
