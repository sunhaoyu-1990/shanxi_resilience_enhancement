# 路径修正与折返识别算法开发计划

## 1. 项目背景

当前业务中，每条通行记录包含：

- `enid`：入口收费站 ID 或入口收费单元 ID
- `exid`：出口收费站 ID 或出口收费单元 ID
- `raw_path`：过程通行路径，由收费单元 ID 使用 `|` 拼接

示例：

```text
G003061001000110|G003061001000210|G003061001000310|G003061001000410|G003061001000510|G003061001003010|G003061001000610|G003061001000710|G003061001000810|G003061001001210|G003061001001310|G003061002000110|G003061002000210|G003061002000310|G300161001000510|G300161001000610|G300161001000710|G300161001000910|G300161001000810|G300161001001110|G300161001001210|G300161001001310|G000561003000110|G300161001001410|G000561003000310|G000561003000410|G000561003000510|G000561003000610|G000561004000110|G000561004000210|G000561004000310|G000561004000410|G000561004000510|G000561004002010|G000561004000610|G000561004000710|G000561005000110|G000561005000210|G000561005000310|G000561005000410|G000561005000510|G000561005000610|G000561005000710|G000561005000810|G000561005000910|G000561005001620
```

现有问题：

1. `raw_path` 存在节点缺失；
2. `raw_path` 存在局部顺序错误；
3. `raw_path` 存在错误节点、非法节点、异常跳点；
4. `raw_path` 中可能存在重复节点、局部环路、掉头或折返；
5. 需要基于拓扑关系表对路径进行尽可能鲁棒的修正；
6. 修正后需要匹配经纬度；
7. 需要计算路径质量指标，判断是否存在多次折返。

本项目目标是开发一套**基于拓扑图的通行路径修正与折返识别算法**。

---

## 2. 核心思想

不要把该问题理解成简单的字符串补全，而应理解为：

> 带噪声观测路径在有向路网拓扑图上的路径重建问题。

其中：

| 信息 | 可信度 | 作用 |
|---|---:|---|
| `enid`、`exid` | 高 | 作为路径起终点强约束 |
| 拓扑关系表 | 高 | 判断路径是否合法、补全缺失段 |
| 原始通行路径 `raw_path` | 中低 | 作为观测证据，可能缺失、乱序、重复、混入错误节点 |
| 经纬度 / 几何信息 | 中高 | 用于路径质量校验、折返识别、空间指标计算 |

算法目标不是简单地让原始相邻节点全部连通，而是：

> 从 `enid` 到 `exid`，在拓扑图中寻找一条最符合原始观测路径、且拓扑合法、折返较少的修正路径。

---

## 3. 总体开发目标

开发一个 Python 模块，输入单条或批量路径记录，输出：

1. 修正后的收费单元路径；
2. 修正后的路径经纬度序列；
3. 路径修正详情；
4. 路径质量评分；
5. 折返识别指标；
6. 修正置信度；
7. 是否需要人工复核。

## 8. 核心流程设计

### 8.1 主函数

在 `repair_pipeline.py` 中实现：

```python
def repair_path(enid: str, exid: str, raw_path: str, graph, node_info, config: dict) -> dict:
    """
    修正单条通行路径。

    Args:
        enid: 入口收费站或收费单元 ID
        exid: 出口收费站或收费单元 ID
        raw_path: 原始路径字符串，节点之间使用 | 拼接
        graph: 拓扑图对象
        node_info: 节点属性字典
        config: 参数配置

    Returns:
        dict: 路径修正结果
    """
```

### 8.2 主流程

```text
1. 解析 raw_path
2. 连续重复去重
3. 过滤非法节点
4. 解析 enid / exid 对应的起终点候选节点
5. 遍历起终点候选组合
6. 对每组固定起终点执行路径修正
7. 选择综合评分最优方案
8. 匹配经纬度
9. 计算路径质量指标
10. 输出修正结果
```

### 8.3 起终点候选组合

```python
def repair_path(enid, exid, raw_path, graph, node_info, config):
    raw_nodes = normalize_raw_path(raw_path)

    start_candidates = resolve_start_candidates(enid)
    end_candidates = resolve_end_candidates(exid)

    best_result = None

    for start_node in start_candidates:
        for end_node in end_candidates:
            result = repair_path_with_fixed_start_end(
                start_node=start_node,
                end_node=end_node,
                raw_nodes=raw_nodes,
                graph=graph,
                node_info=node_info,
                config=config
            )

            if best_result is None or result["total_score"] > best_result["total_score"]:
                best_result = result

    return best_result
```

## 10. 图模型与最短路模块

### 10.1 图模型

文件：`graph_model.py`

建议使用内存邻接表：

```python
class RoadGraph:
    def __init__(self):
        self.out_edges = {}
        self.in_edges = {}
        self.nodes = set()
        self.edge_cost = {}

    def has_node(self, node_id: str) -> bool:
        return node_id in self.nodes

    def has_direct_edge(self, from_node: str, to_node: str) -> bool:
        return to_node in self.out_edges.get(from_node, {})

    def get_out_neighbors(self, node_id: str) -> list[str]:
        return list(self.out_edges.get(node_id, {}).keys())

    def get_edge_cost(self, from_node: str, to_node: str) -> float:
        return self.edge_cost[(from_node, to_node)]
```

### 10.2 最短路模块

文件：`shortest_path.py`

必须实现缓存。

```python
class ShortestPathService:
    def __init__(self, graph):
        self.graph = graph
        self.shortest_path_cache = {}
        self.ksp_cache = {}

    def shortest_path(self, start: str, end: str) -> tuple[list[str] | None, float]:
        """
        返回 start 到 end 的最短路径和成本。
        如果不可达，返回 (None, inf)。
        """

    def k_shortest_paths(self, start: str, end: str, k: int = 5) -> list[tuple[list[str], float]]:
        """
        返回 K 条候选路径。
        如果暂时不实现真正 KSP，可先返回最短路径作为候选。
        后续再替换为 Yen 算法或 pgRouting pgr_ksp。

```

### 10.3 实现建议

第一版可以用 `networkx`：

```python
import networkx as nx
```

构建 `DiGraph`，使用：

```python

nx.shortest_path_length(G, source=start, target=end, weight="cost_m")
```

K 最短路可以先用：

```python

```

注意：`shortest_simple_paths` 对大图可能较慢，因此必须限制：

```python

```

## 11. 局部窗口顺序修正

文件：`local_reorder.py`

### 11.1 背景

原始路径可能出现局部乱序：

```text
A|C|B|D
```

真实路径可能是：

```text
A|B|C|D
```

全局重排风险很高，因此只允许在小窗口内重排。

### 11.2 目标

对长度为 3 到 5 的局部窗口尝试重新排序，使窗口内拓扑连续性更好。

### 11.3 函数设计

```python
def local_reorder_by_topology(nodes: list[str], graph, window_size: int = 4) -> list[str]:
    """
    对路径做局部窗口顺序修正。

    原则：
    1. 只在小窗口内重排；
    2. 只有重排后拓扑连续性明显提升才替换；
    3. 不做全局排序；
    4. 不破坏 enid/exid 强约束。

```

### 11.4 窗口评分

## 12. 可信锚点选择模块

文件：`anchor_selector.py`

### 12.1 定义

锚点是原始路径中较可信的节点。

修正路径会被约束为：

```text
start_node -> anchor_1 -> anchor_2 -> ... -> anchor_k -> end_node
```

### 12.2 候选锚点筛选

对原始路径中的每个节点 `v`，计算：

```text
d_start_v = shortest_cost(start_node, v)
d_v_end = shortest_cost(v, end_node)
d_start_end = shortest_cost(start_node, end_node)

detour_ratio = (d_start_v + d_v_end) / d_start_end
```

如果：

```text
detour_ratio <= max_anchor_detour_ratio
```

则认为该节点可能处于合理路径范围内，可以作为候选锚点。

### 12.3 函数设计

```python
def filter_anchor_candidates(
    start_node: str,
    end_node: str,
    raw_nodes: list[str],
    sp_service,
    config: dict
) -> list[dict]:
    """
    返回候选锚点列表。

    每个锚点结构：
    {
        "node": str,
        "raw_index": int,
        "detour_ratio": float,
        "dist_from_start": float,
        "dist_to_end": float
    }
    """
```

### 12.4 锚点链选择

第一版可以使用简化策略：

1. 候选锚点按 `raw_index` 排序；
2. 从前向后扫描；
3. 只保留可以从上一个锚点可达的节点；
4. 如果一个节点导致明显绕行或不可达，则跳过。

第二版升级为动态规划：

```python
def select_anchors_by_dp(
    start_node: str,
    end_node: str,
    candidate_anchors: list[dict],
    sp_service,
    config: dict
) -> list[dict]:
    """
    使用动态规划选择最优锚点链。
    """
```

动态规划转移代价包括：

```text
- 分段最短路长度
- 跳过原始节点数量
- 绕行惩罚
- 方向反转惩罚
- 折返惩罚
```

### 13.1 功能


### 13.3 分段逻辑

对于每一对相邻锚点：

```text
anchor_i -> anchor_{i+1}
```

执行：

1. 获取原始路径中两个锚点之间的观测片段；
2. 查询 K 条候选路径；
3. 对每条候选路径计算分段评分；
4. 选择评分最优的候选路径；
5. 拼接到最终路径中。

### 13.4 观测片段获取

```python
def get_observed_slice(raw_nodes: list[str], anchor_a: dict, anchor_b: dict) -> list[str]:
    """
    获取两个锚点在原始路径索引之间的观测节点片段。
    """
```

### 13.5 分段评分

```python
def segment_score(candidate_path: list[str], observed_slice: list[str], graph, config: dict) -> float:
    """
    分数越低越好。
    """
```

建议评分：

```text
segment_score =
    0.40 * 路径长度归一化
  + 0.30 * 原始节点不匹配惩罚
  + 0.15 * 插入节点数量惩罚
  + 0.10 * 折返惩罚
  + 0.05 * 跳过节点惩罚
```

其中：

```text
原始节点不匹配惩罚 = 1 - LCS(candidate_path, observed_slice) / len(observed_slice)
```

```python
def lcs_length(a: list[str], b: list[str]) -> int:
    """计算最长公共子序列长度。"""
```

## 14. 后处理模块

文件：`postprocess.py`

### 14.1 功能

修正后的路径可能仍存在局部小环、连续重复、明显异常重复段，需要做后处理。

### 14.2 函数设计

```python
        {
            "nodes": [...],
            "removed_loops": [...],
            "removed_duplicates": [...]
        }
```

### 14.3 后处理策略

1. 再次去除相邻重复节点；
2. 对 `A -> B -> A` 形式的小环进行识别；
3. 对短距离重复震荡段进行压缩；
4. 允许一次疑似掉头，不强制删除；
5. 多次重复折返时标记为低置信度，而不是盲目删除。

### 16.1 需要计算的指标

```python
def calc_detour_ratio(path: list[str], start_node: str, end_node: str, graph, sp_service) -> float:
    """
    corrected_path_length / shortest_path_length(start_node, end_node)
    """
```

### 16.7 综合折返指数

```python
def calc_backtrack_index(metrics: dict, config: dict) -> float:
    """
    计算 0~100 的折返指数。
    """
```

建议公式：

```text
BTI =
    30 * reverse_score
  + 25 * backward_score
  + 20 * repeat_score
  + 15 * uturn_score
  + 10 * detour_score
```

示例实现：

```python
def calc_backtrack_index(metrics: dict, config: dict) -> float:
    reverse_score = min(metrics["reverse_edge_count"] / 2, 1)
    backward_score = min(metrics["backward_progress_count"] / 3, 1)
    repeat_score = min(metrics["repeated_node_count"] / 3, 1)
    uturn_score = min(metrics["u_turn_count"] / 2, 1)
    detour_score = min(max(metrics["detour_ratio"] - 1.2, 0) / 1.0, 1)

    return (
        30 * reverse_score
        + 25 * backward_score
        + 20 * repeat_score
        + 15 * uturn_score
        + 10 * detour_score
    )
```

### 16.8 折返指数分级

| `backtrack_index` | 判断 |
|---:|---|
| 0 - 20 | 路径正常，基本无折返 |
| 20 - 40 | 轻微异常，可能有一次掉头或互通绕行 |
| 40 - 60 | 明显异常，需要复核 |
| > 60 | 多次折返或路径修正失败概率高 |

## 22. 关键实现注意事项

### 22.1 不要全局去重

错误做法：

```python
path = list(set(path))
```

原因：

1. 会破坏路径顺序；
2. 会误删真实掉头；
3. 会掩盖折返问题。

只能先做相邻重复去重。

---

### 22.2 不要盲目相信原始路径顺序

原始路径可能乱序，所以：

```text
A|C|B|D
```

不应简单认为 `C -> B` 是真实通行。

需要通过拓扑可达性和整体评分判断。

---

### 22.3 不要只依赖经纬度判断折返

高速互通、匝道、立交附近可能出现大角度转向。  
几何掉头指标必须和拓扑指标结合使用。

---

### 22.4 起终点是强约束

修正后的路径必须满足：

```text
corrected_path[0] = start_node
corrected_path[-1] = end_node
```

如果不能满足，状态应为失败或低置信度。

---

### 22.5 路径修正结果必须可解释

必须输出：

1. 哪些节点被删除；
2. 哪些节点被插入；
3. 选择了哪些锚点；
4. 每一段怎么补全；
5. 为什么判定为高置信度或低置信度。

## 26. 总结

本项目的核心不是简单补全字符串，而是实现一套面向高速路网拓扑的路径重建算法。

推荐实现路线：

```text
V1：相邻补全 + 异常跳点
V2：可信锚点 + 分段 K 最短路 + LCS 匹配
V3：局部重排 + 折返增强 + 批量质量统计
```

最终算法应实现：

```text
起终点强约束
原始路径弱观测
拓扑合法性修正
经纬度匹配
折返指标计算
置信度评分
人工复核标记
```

这套方案可以适应路径缺失、顺序错误、错误节点、重复节点、局部折返、起终点候选歧义等复杂情况，并能为后续 OD 路径分析、施工影响评估、分流路径优化等模块提供可靠的路径数据底座。
