# 收费单元路径数据 - 建表与数据导入指南

## 概述

本指南说明如何快速创建 `dwd_section_path` 表并批量导入 6 个版本的收费单元唯一路径数据。

## 文件说明

| 文件 | 位置 | 说明 |
|------|------|------|
| 数据库初始化脚本 | `sql/init_database.sql` | 创建 schema、所有表、插入版本配置 |
| 数据导入脚本 | `scripts/import_section_path.py` | Python 脚本，读取 Excel 并写入 PostgreSQL |
| DDL - 版本配置表 | `sql/ddl/dim/create_dim_section_path_version.sql` | 版本配置表定义 |
| DDL - 路径数据表 | `sql/ddl/dwd/create_dwd_section_path.sql` | 收费单元路径明细表定义 |
| 数据文件 | `research/data/基础数据/2024-2026年收费单元唯一路径/` | 6 个版本的 Excel 数据 |

## 前置条件

1. ✅ PostgreSQL 15 + PostGIS 3 已安装并运行
2. ✅ 数据库 `shanxi_resilience_db` 已创建
3. ✅ 环境变量已配置（`.env` 文件）
4. ✅ Python 虚拟环境已激活
5. ✅ 依赖包已安装（`uv sync` 或 `pip install`）

## 快速开始

### 第一步：配置环境变量

确保 `.env` 文件中配置了正确的数据库连接信息：

```env
# 数据库配置
DATABASE_HOST=192.168.0.75
DATABASE_PORT=5432
DATABASE_USER=sunhaoyu
DATABASE_PASSWORD=your_password
DATABASE_DBNAME=shanxi_resilience_db
DATABASE_SCHEMA=shanxi_resilience
```

### 第二步：初始化数据库

使用 `psql` 执行初始化脚本：

```bash
# 方式一：直接执行完整初始化脚本
psql -h 192.168.0.75 -U sunhaoyu -d shanxi_resilience_db -f sql/init_database.sql

# 方式二：分步执行（推荐用于调试）
psql -h 192.168.0.75 -U sunhaoyu -d shanxi_resilience_db

# 在 psql 中：
\i sql/ddl/dim/create_dim_section_path_version.sql
\i sql/ddl/dwd/create_dwd_section_path.sql
-- 插入版本配置数据（手动执行或用脚本）
```

### 第三步：验证表结构

```bash
# 连接数据库
psql -h 192.168.0.75 -U sunhaoyu -d shanxi_resilience_db

# 查看表
\dt shanxi_resilience.

# 查看版本配置
SELECT * FROM shanxi_resilience.dim_section_path_version ORDER BY version_yyyyMM;

# 查看表结构
\d shanxi_resilience.dwd_section_path
```

### 第四步：导入数据

#### 4.1 先查看可用版本

```bash
python scripts/import_section_path.py --list
```

#### 4.2 预览数据（不写入）

```bash
# 预览所有版本
python scripts/import_section_path.py --all --preview

# 预览指定版本
python scripts/import_section_path.py --version 202401 --preview
```

#### 4.3 执行导入

```bash
# 导入所有版本（增量，跳过已存在的）
python scripts/import_section_path.py --all

# 导入所有版本（覆盖）
python scripts/import_section_path.py --all --overwrite

# 导入指定版本
python scripts/import_section_path.py --version 202401,202409

# 导入指定版本并覆盖
python scripts/import_section_path.py --version 202401 --overwrite
```

### 第五步：验证数据

```sql
-- 查看各版本数据量
SELECT version_yyyyMM, COUNT(*) AS cnt
FROM shanxi_resilience.dwd_section_path
GROUP BY version_yyyyMM
ORDER BY version_yyyyMM;

-- 查看 sample 数据
SELECT * FROM shanxi_resilience.dwd_section_path
WHERE version_yyyyMM = '202401'
LIMIT 5;

-- 检查数据质量
SELECT
    version_yyyyMM,
    COUNT(*) AS total,
    COUNT(*) FILTER (WHERE id IS NULL) AS null_id,
    COUNT(*) FILTER (WHERE DIRECTION NOT IN (1, 2)) AS invalid_direction
FROM shanxi_resilience.dwd_section_path
GROUP BY version_yyyyMM;
```

## 数据导入脚本详解

### 命令行参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--all` | 导入所有可用版本 | `--all` |
| `--version` | 导入指定版本（逗号分隔） | `--version 202401,202409` |
| `--overwrite` | 覆盖已存在的版本数据 | `--overwrite` |
| `--preview` | 预览模式，不写入数据库 | `--preview` |
| `--list` | 列出所有可用版本 | `--list` |

1. **自动数据类型转换**
   - 日期字段自动解析
   - 数值字段类型转换
   - 空值处理

2. **数据质量校验**
   - 必填字段检查
   - 主键唯一性检查
   - 枚举值范围验证

3. **批量写入**
   - 分块写入（每批 1000 条）
   - 支持增量/覆盖模式
   - 事务保障

4. **详细日志**
   - 每个版本的处理进度
   - 数据校验结果
   - 错误信息

## 常见问题

### Q: 如何只重新导入某个版本？

```bash
python scripts/import_section_path.py --version 202401 --overwrite
```

### Q: 导入时提示"版本文件不存在"？

检查文件名格式是否正确：`{YYYYMM}单元唯一路径.xlsx`

### Q: 如何查看导入日志？

日志配置在 `configs/logging.yaml`，默认输出到控制台和 `logs/` 目录。

### Q: 数据导入很慢怎么办？

- 确保数据库连接正常
- 检查网络带宽
- 脚本已使用批量写入（每批 1000 条），性能应该良好

### Q: 如何在 Python 代码中调用导入功能？

```python
from scripts.import_section_path import SectionPathImporter

importer = SectionPathImporter()

# 导入单个版本
result = importer.import_version("202401", overwrite=True)

# 导入所有版本
results = importer.import_all(overwrite=True)
```

## 维护

### 新增版本

当有新版本数据时：

1. 将文件放入 `research/data/基础数据/2024-2026年收费单元唯一路径/`
2. 文件名格式：`{YYYYMM}单元唯一路径.xlsx`
3. 在 `dim_section_path_version` 表中插入版本配置
4. 运行导入脚本：`python scripts/import_section_path.py --version {YYYYMM}`

### 数据更新

如需更新已存在版本的数据：

```bash
python scripts/import_section_path.py --version {YYYYMM} --overwrite
```
