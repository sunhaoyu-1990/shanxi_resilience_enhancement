# M6 批量运行说明

## 功能概述

M6 OD-Section-Path 映射批量运行，支持：
- 多版本连续处理（202512 ~ 202603 等）
- Hive 数据库自动切换（2025年表用 `dbbase2025`，2026年表用 `dbbase2026`）
- checkpoint 断点续跑（进程崩溃后自动从断点恢复）
- 每批次实时 upsert 到数据库，内存占用小（约 5MB/批次）
- rank 在所有批次、所有版本完成后统一计算

## 数据库切换规则

| 数据版本 | Hive 数据库 | 拓扑版本 | section_version |
|---------|------------|---------|----------------|
| 202512 及之前 | `dbbase2025` | `202507` | 同数据版本 |
| 202601 及之后 | `dbbase2026` | `202512` | 同数据版本 |

## 入口文件

| 文件 | 用途 |
|------|------|
| `src/jobs/run_m6.py` | 单版本运行入口 |
| `src/jobs/run_m6_batch.py` | 多版本批量运行入口 |

## 快速开始

### 1. 查看当前运行状态

```bash
uv run python -m src.jobs.run_m6_batch --status
```

输出示例：
```
版本         表名                                      offset    processed status       topo
----------------------------------------------------------------------------------------------------
202603     gstx_exit_with_min_fee202603               200          200 completed    202512
202602     gstx_exit_with_min_fee202602               0             0 running       202512
```

### 2. 多版本批量运行

```bash
# 处理 202512 ~ 202603（4个版本）
uv run python -m src.jobs.run_m6_batch \
    --start-version 202512 \
    --end-version 202603 \
    --batch-size 500000 \
    --workers 2
```

### 3. 单版本运行

```bash
# 全量模式（处理全部 ~39M 条记录）
uv run python -m src.jobs.run_m6 \
    --version 202603 \
    --batch-size 500000 \
    --workers 2

# 测试模式（只处理 batch-size 条，用于验证）
uv run python -m src.jobs.run_m6 \
    --version 202603 \
    --batch-size 100 \
    --save-local
```

### 4. 断点续跑

中途断后，直接重新运行相同命令即可自动跳过已完成的版本：

```bash
uv run python -m src.jobs.run_m6_batch \
    --start-version 202512 \
    --end-version 202603 \
    --batch-size 500000 \
    --workers 2
```

输出中会看到：
```
版本 202603 已完成，跳过（status=completed）
开始处理版本: 202602
checkpoint 续跑，从 offset=500000 继续...
```

### 5. 重置版本（从头重跑）

```bash
uv run python -m src.jobs.run_m6_batch --reset-version 202603
```

会重置 checkpoint 的 `offset=0`，`status=running`，下次运行会从头开始。

### 6. 仅计算 rank（漏算时补跑）

```bash
# 单版本
uv run python -m src.jobs.run_m6_batch --compute-rank --version 202603

# 所有版本
uv run python -m src.jobs.run_m6_batch --compute-rank
```

## 参数说明

### run_m6_batch.py 参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--start-version` | 起始版本（如 202512） | 无 |
| `--end-version` | 结束版本（如 202603） | 无 |
| `--version` | 单版本运行（与 start/end 互斥） | 无 |
| `--batch-size` | 每批处理记录数 | 500000 |
| `--workers` | 并行 worker 进程数（2GB RAM 建议用 2） | 2 |
| `--status` | 查看 checkpoint 状态 | False |
| `--reset-version` | 重置指定版本的 checkpoint | 无 |
| `--compute-rank` | 仅计算 rank 并派生 map | False |

### run_m6.py 参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--version` | 版本年月 | 202603 |
| `--hive-table` | Hive 表名（默认自动生成） | `gstx_exit_with_min_fee{version}` |
| `--section-version` | dwd_section_path 版本 | 同 version |
| `--topo-version` | 拓扑版本 | 202512 |
| `--batch-size` | 每批处理记录数 | 500000 |
| `--workers` | 并行 worker 数 | 2 |
| `--save-local` | 保存结果到本地 JSON（测试用） | False |
| `--output-dir` | 本地输出目录 | `outputs/m6_test` |

## 数据流程

```
Hive 表分批读取
    ↓
每批次：intervalgroup 拓扑修复（并行）→ section 映射 → 聚合
    ↓
每批次结果实时 upsert 到 dwd_od_section_path_numpath_freq（ig_count 累加）
    ↓
每批次后更新 checkpoint（offset, processed, last_batch_time）
    ↓
所有版本、所有批次完成后
    ↓
计算 ig_rank（按 ig_count 降序）
    ↓
从 freq 表派生 dwd_od_section_path_map（取 best fixed_ig）
```

## 输出表说明

| 表名 | 说明 | 主键 |
|------|------|------|
| `dwd_od_section_path_numpath_freq` | 每批次实时 upsert，ig_count 累加 | `(enid, exid, numpath, fixed_intervalgroup, version_yyyyMM, topo_version)` |
| `dwd_od_section_path_map` | 所有批次完成后统一派生 | `(enid, exid, numpath, version_yyyyMM)` |
| `m6_checkpoint` | 断点续跑记录 | `(table_name, version_yyyyMM)` |

## checkpoint 状态说明

| status | 含义 |
|--------|------|
| `running` | 处理中，可断点续跑 |
| `completed` | 已完成，运行时会自动跳过 |
| `failed` | 失败（暂未使用） |

## 验证结果

运行后可用 SQL 核对数据：

```sql
-- 查看 freq 表整体统计
SELECT
    COUNT(*) AS total_records,
    COUNT(DISTINCT enid || '|' || exid) AS od_pair_count,
    COUNT(DISTINCT numpath) AS numpath_count,
    SUM(ig_count) AS total_ig_count,
    AVG(ig_count) AS avg_ig_count
FROM dwd_od_section_path_numpath_freq
WHERE version_yyyyMM = '202603';

-- 查看 map 表
SELECT enid, exid, numpath, fixed_intervalpath,
       intervalpath_cnt, total_trip_cnt, path_freq_ratio
FROM dwd_od_section_path_map
WHERE version_yyyyMM = '202603'
ORDER BY enid, exid, numpath;

-- 查看一致性分布
SELECT
    CASE
        WHEN path_freq_ratio >= 0.9 THEN '高一致性(>=0.9)'
        WHEN path_freq_ratio >= 0.7 THEN '中一致性(0.7-0.9)'
        ELSE '低一致性(<0.7)'
    END AS consistency,
    COUNT(*) AS cnt
FROM dwd_od_section_path_map
WHERE version_yyyyMM = '202603'
GROUP BY 1
ORDER BY 1;

-- 验证 checkpoint 状态
SELECT table_name, version_yyyyMM, batch_offset,
       records_processed, status, topo_version
FROM m6_checkpoint
ORDER BY version_yyyyMM;
```

## 批次日志

每批次处理后会保存 JSON 日志到：

```
src/modules/m6_od_section_path/logs/v{version}/batch_{batch_no:04d}.json
```

日志内容：
- `input_records`: 批次读取记录数
- `valid_records`: 有效记录数（处理后）
- `ig_counts`: 每个 `(enid, exid, numpath, fixed_ig)` 的 count
- `db_freq_written`: 写入 DB 的 freq 记录数

用于核对批次过程数据和 DB 实际数据是否一致。

## 故障处理

### 进程中途崩溃

直接重新运行相同命令即可，会自动从 checkpoint 记录的 offset 继续。

### 某版本一直失败

```bash
# 重置该版本，从头重跑
uv run python -m src.jobs.run_m6_batch --reset-version 202603

# 然后重新运行
uv run python -m src.jobs.run_m6_batch \
    --start-version 202512 \
    --end-version 202603 \
    --batch-size 500000 \
    --workers 2
```

### rank 计算遗漏

```bash
# 补算单个版本
uv run python -m src.jobs.run_m6_batch --compute-rank --version 202603

# 补算所有版本
uv run python -m src.jobs.run_m6_batch --compute-rank
```

### 数据验证失败

```bash
# 用测试模式跑小批量，保存本地 JSON 对比
uv run python -m src.jobs.run_m6 \
    --version 202603 \
    --batch-size 100 \
    --save-local

# 查看输出
cat outputs/m6_test/m6_test_v202603.json | python -m json.tool | less
```
