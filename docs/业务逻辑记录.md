### BL-001: 判断收费单元是否属于交控集团

**编号**: BL-001

**名称**: 判断收费单元是否属于交控集团

**创建日期**: 2026-04-13

**业务描述**:
根据收费单元 ID 判断该收费单元是否属于交控集团。

---

### 详细逻辑

#### 输入
- `section_id` - 收费单元 ID（字符串）
- 参考数据文件：`research/data/收费路段.xls`

#### 处理步骤

1. **提取收费单元 ID 前 11 位**
   - 从 `section_id` 中提取前 11 位字符
   - 例如：`section_id = "ABC123456789XYZ"` → 提取 `"ABC1234567"`

2. **匹配收费路段编号**
   - 使用提取的 11 位字符
   - 在 `research/data/收费路段.xls` 文件中查找
   - 匹配字段：`收费路段编号`

3. **判断路段性质**
   - 找到匹配的记录后，查看 `路段性质` 字段
   - 如果 `路段性质` = `"还贷性"` → **属于交控集团**
   - 其他情况（包括未匹配到）→ **不属于交控集团**

#### 输出
- `is_shanjiao` - 布尔值
  - `true` - 属于交控集团
  - `false` - 不属于交控集团

---

### 参考数据说明

**文件位置**: `research/data/收费路段.xls`

**关键字段**:
| 字段名 | 说明 | 示例 |
|--------|------|------|
| `收费路段编号` | 收费路段编号（用于匹配） | `"ABC1234567"` |
| `路段性质` | 路段性质（用于判断） | `"还贷性"` |

---

### 使用场景

此逻辑在以下模块中使用：

| 模块 | 用途 |
|------|------|
| **M0 数据工程** | 收费单元属性标记 |
| **M1 通行能力评估** | 仅计算交控集团收费单元 |
| **M2 流量与 OD 补全** | 根据业务需求筛选 |
| **其他模块** | 根据业务需求筛选 |

---

### 伪代码示例

```python
def is_shanjiao_section(section_id: str, toll_segments_df: DataFrame) -> bool:
    """
    判断收费单元是否属于交控集团

    Args:
        section_id: 收费单元 ID
        toll_segments_df: 收费路段数据 DataFrame

    Returns:
        bool: True=交控, False=非交控
    """
    # 1. 提取前 11 位
    if len(section_id) < 11:
        return False

    segment_prefix = section_id[:11]

    # 2. 匹配收费路段编号
    matched = toll_segments_df[
        toll_segments_df["收费路段编号"] == segment_prefix
    ]

    if matched.empty:
        return False

    # 3. 判断路段性质
    segment_type = matched.iloc[0]["路段性质"]

    return segment_type == "还贷性"
```

### SQL 示例（如果数据已入库）

```sql
-- 判断收费单元是否属于交控集团
WITH section_prefix AS (
    SELECT
        section_id,
        LEFT(section_id, 11) AS segment_prefix
    FROM dwd_section_info
),
matched_segment AS (
    SELECT
        sp.section_id,
        ts.路段性质
    FROM section_prefix sp
    LEFT JOIN dim_toll_segment ts
        ON sp.segment_prefix = ts.收费路段编号
)
SELECT
    section_id,
    CASE
        WHEN 路段性质 = '还贷性' THEN true
        ELSE false
    END AS is_shanjiao
FROM matched_segment;
```

## BL-002: 收费单元唯一路径版本管理

**编号**: BL-002

**名称**: 收费单元唯一路径版本管理

**创建日期**: 2026-04-13

**业务描述**:
收费单元唯一路径信息会随时间更新，有多个版本文件。需要根据数据日期选择对应的版本文件。

**数据字典**: `research/data/基础数据/2024-2026年收费单元唯一路径/数据字典.xlsx`

---

### 版本选择逻辑

#### 输入
- `data_date` - 数据日期（YYYY-MM-DD 格式）

#### 处理逻辑

1. **提取年月**
   - 从 `data_date` 中提取年月，格式为 `YYYYMM`
   - 例如：`data_date = "2024-05-15"` → `"202405"`

2. **查找适用版本**
   - 找到所有版本文件的生效年月 ≤ 目标年月
   - 选择其中最大的（最新的）版本
   - 例如：目标年月 `"202405"`，适用版本是 `"202401"`

3. **确定文件路径**
   - 根据选定的版本年月，构造文件名
   - 例如：版本 `"202401"` → 文件 `"202401单元唯一路径.xlsx"`

#### 输出
- `version_yyyyMM` - 适用版本年月（YYYYMM 格式）
- `file_path` - 适用版本文件的完整路径

---

### 伪代码示例

```python
from pathlib import Path
from datetime import datetime
from typing import Tuple

# 版本配置
VERSION_CONFIG = {
    "202401": "2024-01-01",
    "202409": "2024-09-01",
    "202412": "2024-12-01",
    "202507": "2025-07-01",
    "202512": "2025-12-01",
    "202603": "2026-03-01",
}

DATA_DIR = Path("research/data/基础数据/2024-2026年收费单元唯一路径")


def get_section_path_version(data_date: str) -> Tuple[str, Path]:
    """
    根据数据日期获取适用的收费单元唯一路径版本

    Args:
        data_date: 数据日期，格式 YYYY-MM-DD

    Returns:
        (version_yyyyMM, file_path): 版本年月和文件路径
    """
    # 1. 解析日期
    dt = datetime.strptime(data_date, "%Y-%m-%d")
    target_yyyymm = dt.strftime("%Y%m")

    # 2. 找到所有生效年月 <= 目标年月的版本
    applicable_versions = [
        v for v in VERSION_CONFIG.keys()
        if v <= target_yyyymm
    ]

    if not applicable_versions:
        raise ValueError(f"没有找到适用于日期 {data_date} 的版本")

    # 3. 选择最新的适用版本
    selected_version = max(applicable_versions)

    # 4. 构造文件路径
    file_name = f"{selected_version}单元唯一路径.xlsx"
    file_path = DATA_DIR / file_name

    if not file_path.exists():
        raise FileNotFoundError(f"版本文件不存在: {file_path}")

    return selected_version, file_path
```

### SQL 查询示例：根据日期获取适用版本

```sql
-- 查询某个日期适用的版本
WITH target_date AS (
    SELECT '2024-05-15'::DATE AS dt
),
target_yyyymm AS (
    SELECT TO_CHAR(dt, 'YYYYMM') AS yyyymm
    FROM target_date
)
SELECT
    v.*
FROM dim_section_path_version v
CROSS JOIN target_yyyymm t
WHERE v.version_yyyyMM <= t.yyyymm
ORDER BY v.version_yyyyMM DESC
LIMIT 1;
```
