# M8 路径修正模块 — 实施计划

> 创建时间：2026-05-21
> 状态：Phase 1 已完成，Phase 2 已完成

## 项目背景

基于 GPT5.5 生成的《路径修正与折返识别算法开发计划》（`docs/路径修正与折返识别算法开发计划.md`），将其适配为项目标准模块格式，作为独立模块 M8 开发。

## 核心设计决策

1. **不引入 networkx**：所有最短路径走已有 pgRouting SQL
2. **复用 M2 TopologyChecker 思路**：封装为 `RoadGraph` 类
3. **模块独立**：不依赖 M0-M7 运行结果
4. **V1 相邻补全 → V2 锚点 DP + 分段 KSP**：渐进式开发

## Phase 2：V2 锚点 DP + 分段 KSP ✅ 已完成

### V2 算法流程

```
1. 路径标准化（同 V1）
2. 候选锚点筛选：detour_ratio = (d_start_v + d_v_end) / d_start_end
3. DP 选择最优锚点链（分段成本 + 跳过惩罚 + 绕行惩罚）
4. 分段 K 最短路拼接：
   a. 对每对相邻锚点查询 K 条候选路径（pgRouting pgr_ksp）
   b. LCS 观测匹配评分 + 插入/跳过/折返惩罚
   c. 选择评分最优候选拼接
5. 后处理（相邻去重、小环识别、允许一次掉头）
6. 若 V2 不可行则回退 V1
```
