====================
一、项目背景与当前范围
====================

当前一期项目的主链路固定为：

- M0 数据工程
- M1 通行能力评估
- M2 流量与OD迁移统计补全
- M3 交通影响分析
- M4 分流路径优化
- M5 通行费影响测算

当前阶段强调：
- 先做最小可用版本（MVP）
- 先搭工程骨架，不追求复杂业务逻辑终版
- 以统计、经验规则、工程化实现为主
- 不擅自扩展到高阶研究闭环
- 不擅自改变模块编号和边界

当前已明确的重要约束：
- 技术栈：Python + SQL 为主
- 数据库：PostgreSQL + PostGIS
- 模式：离线批处理为主，查询服务为辅
- 配置：YAML + .env
- 文档：Org
- 核心风格：SQL 承担核心统计与分析逻辑，Python 负责编排、配置、日志、校验、服务封装
- 收费单元维表必须纳入字段：section_number
- 当前路网拓扑基础结构为：本收费单元ID -> 下一个收费单元ID（一对多）

注意：
当前阶段不要臆造未确认的业务规则。
未确认项允许使用 TODO 标记，但工程结构必须完整。
