* 项目简介
本项目用于支撑“多路段改扩建及养护工程并发施工条件下”的通行服务保障分析与测算。
一期目标是先完成数据算法层面的最小可用版本（MVP），形成可运行的后端算法主链路，为施工方案分析、分流路径生成、通行费影响测算提供基础能力。

当前工程定位：
- 以 Python + SQL 为主
- 以离线批处理任务为主
- 以 PostgreSQL + PostGIS 为主数据库
- 以查询服务为辅，不做在线复杂重算
- 以“先可交付、后增强”为原则推进

* 一期范围
当前一期主链路包括以下模块：
- M0 数据工程
- M1 通行能力评估
- M2 流量与OD迁移统计补全
- M3 交通影响分析
- M4 分流路径优化
- M5 通行费影响测算

说明：
- 当前版本以统计、经验规则、工程化实现为主
- 当前版本不追求完整研究闭环
- 高阶寻优、效果评估、动态优化等能力后续再逐步增强

* 一期目标输出
一期重点输出包括但不限于：
- 数据底座与标准化数据表
- 收费单元施工映射结果
- 施工期间收费单元可通行当量
- 施工影响下的OD迁移与流量修正结果
- 候选绕行路径及分流管控单元
- 通行费增收/减收测算结果
- 最优绕行方案

* 工程原则
- 先定表，再定逻辑，再写代码
- SQL 承担核心统计和落表逻辑
- Python 负责参数、编排、日志、校验
- 工程代码与研究代码严格分离
- 所有输出表必须先登记口径
- 所有模块必须先明确输入表、输出表、中间表

* 当前关键基础约束
- 收费单元单一路径标号字段 = section_number
- section_number 是后续路径合并、收费单元归并、断面路径映射的重要基础字段
- 路网拓扑当前按“本收费单元ID -> 下一个收费单元ID（一对多）”方式接入

* 开发启动顺序
1. 建立项目目录结构
2. 完成基础配置与连接
3. 完成表口径总表
4. 完成模块输入输出定义
5. 编写 DDL / 基础 SQL
6. 编写离线任务编排代码
7. 开展模块联调
8. 接入查询服务

* 核心模块说明

| 模块 | 名称 | 主要职责 |
|------|------|----------|
| M0 | 数据工程 | ODS 数据加载、DIM 维表构建、方案映射、OD-路径映射 |
| M1 | 通行能力评估 | 车道数折算、可用车道数计算、能力规则匹配 |
| M2 | 流量与OD补全 | OD 流量统计、路段流量统计、SourceFlag 溯源 |
| M3 | 交通影响分析 | 供需比计算、影响流量统计、影响等级判定 |
| M4 | 分流路径优化 | 候选路径生成、里程差/收费差计算、分流管控点 |
| M5 | 通行费影响测算 | 通行费增减计算、影响类型判定、方案汇总 |

* 数据层级说明

| 层级 | 用途 | 说明 |
|------|------|------|
| ODS | 原始数据层 | 外部数据直接入库 |
| DIM | 维表层 | 路段、工程方案等维度信息 |
| DWD | 明细事实层 | 路段能力、OD 流量等原子事实 |
| DWS | 汇总事实层 | 按日期/路段/方案等维度的轻度汇总 |
| ADS | 应用层 | 直接可用的分析结果 |
